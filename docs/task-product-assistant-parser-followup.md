# Task: доробити евристичний парсер асистента карточки продукту

**Статус:** хомути / DIN / W — реалізовано в `heuristicProductSpecs.js` (2026); інші класи товарів — за потреби додавати окремі гілки.  
**Контекст:** скрін «Нова карточка продукту», поле **Тип / найменування** = `Хомут нерж 16 - 25`.

## Проблема

- Для такого найменування асистент показує лише загальний рядок **«Числові параметри з назви (уточніть призначення): 16 — 25»**.
- Для користувача очевидніше трактування: **хомут (часто нержавіючий) з діапазоном обжиму діаметра ~16–25 мм** (типова маркування в КТЗ/каталогах).
- Зараз логіка **кріплення** (діаметр × довжина) не спрацьовує, бо пара 16–25 не проходить евристику «довжина ≫ діаметр», і ключове слово **хомут** не обробляється окремо.

## Ціль

Розширити `backend/heuristicProductSpecs.js` (і за потреби ключові слова), щоб **універсально** покривати ще один клас товарів — **хомути / затискачі** — і видавати зрозумілі поля замість сирого «16 — 25».

## Пропозиція реалізації

1. Додати контекст **`HOSE_CLAMP_RE`** (укр./лат.: хомут, worm clamp, hose clamp тощо; опційно «нерж», «нержав», `stainless`, `AISI` у назві).
2. Якщо контекст хомут і знайдено пару **два числа через дефіс** у розумному діапазоні (наприклад 8–200, **друге ≥ перше**, різниця не обов’язково кратна 3 як у дюбеля):
   - Додати характеристики на кшталт:
     - **«Діапазон діаметра обжиму (з назви), мм»** → `16–25` або `16 — 25`;
     - за наявності **нерж** у тексті — **«Матеріал (з назви)»** → «нержавіюча сталь (евристика)».
3. **Не** застосовувати до такої пари ту саму гілку, що для дюбеля (L/D у мм), щоб уникнути конфліктів.
4. Порядок пріоритету в парсері: спочатку специфічні контексти (хомут), потім кріплення, потім загальна пара «уточніть призначення».
5. Тести вручну через `node` або короткий список прикладів у коментарі до модуля:
   - `Хомут нерж 16 - 25`
   - `хомут 12-20`
   - переконатися, що `Дюбель 120-8` лишається Ø × L.

## Опційно (пізніше)

- Якщо знову часто потрапляємо в **mock**: окремо розглянути джерело окрім Вікіпедії (не входить у мінімальний scope цієї задачі).
