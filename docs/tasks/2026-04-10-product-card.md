# Таска: карточка продукту (номенклатура)

**Дата:** 2026-04-10 (завтра)  
**Контекст:** Зараз тип/виробник/категорія живуть у записі `Equipment`; завсклад не може правити чужі склади, але єдиного довідника «продукт» немає.

## Цілі (чернетка)

- [ ] Модель **ProductCard** (або аналог): зв’язок з `categoryId`, канонічні `type` / `manufacturer`, можливо одиниці, ознаки ЗІП/товар.
- [ ] Надходження / імпорт: вибір існуючої карточки або створення нової + **підказки схожих** (уже частково є на імпорті Excel).
- [ ] `Equipment` тримає `productId` (опційно денорм поля для пошуку).
- [ ] Редагування «канонічних» полів номенклатури — лише з карточки / ролі адмін; з позиції складу — лінк, не дубль тексту.
- [ ] Видалення карточки: лише якщо **сума залишків по всіх складах = 0** (перевірка по `productId` або по type+category — уточнити).

## Примітки

- Погодити міграцію існуючих `Equipment` → `productId` (скрипт / етапами).
- Узгодити з імпортом 1С (`stockXlsxImport`, `stockimport_nomenclature_maps`).
