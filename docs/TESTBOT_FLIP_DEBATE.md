# TESTBOT — Дебат 9 AI: мінус → плюс без зміни динаміки ставок

> Дата: 16.07.2026  
> Об'єкт: `FX_TESTBOT_*` sim-журнал (панель live)  
> Обмеження: **не змінювати динаміку ставок** — той самий `riskPerTradePct`, той самий розрахунок `units`, ті самі `maxOpen` / entry interval / pair cooldown.  
> Дозволено: полярність сигналу, фільтри «брати / не брати», виходи (TP/SL у $), метадані журналу.

### Математика (без зміни сайзингу)

```
Avg win (ціль)     ≈ +$1.00
Avg loss (факт)    ≈ −$3.14   (−738.83 / 236, якщо 2 виграші ≈ +$1)
Break-even WR      ≈ 76%      (потрібно при такому R:R)
Факт WR            ≈ 0.84%
Expectancy/угода   ≈ −$3.10   → системний мінус
```

**Висновок панелі:** score/conviction **не розділяє** win/loss. Сигнал зараз **антикорельований** з рухом ціни на горизонті testbot-виходу.

## Раунд 1 — Діагноз (кожен)

### 1. NOVA (Quant)
> *«Це не поганий R:R при хорошому WR. Це майже ідеальний анти-сигнал.»*

| # | Теза |
|---|------|
| N1 | При avgW=$1 / avgL≈$3.1 break-even ≈ **76% WR**. Маємо **<1%** → edge від’ємний на порядок. |
| N2 | Naive **invert replay** (ті самі 238 угод, flip side, виходи лишаються $target / $stop): колишні 236 стопів стають шляхами **в плюс** (ціна йшла проти старої позиції → за новою). Кап прибутку = **+$1**, кап збитку на 2 колишніх wins ≈ **−$4**. Орієнтовно: `236×(+1) + 2×(−4) ≈ +$228` на тій самій серії. |
| N3 | Invert **не змінює** `calcUnitsForRisk` — лише `flipAction`. Це легально під обмеженням. |
| N4 | Після invert WR стане високим, але **R:R лишається поганим** (+1 vs −4). Плюс буде, але крихкий: кілька серій SL з’їдять багато маленьких TP. |

### 2. LENS (Journal Auditor)
> *«238 угод — достатньо для вироку “напрямок ламаний”, недостатньо для тонкого тюнінгу без shadow.»*

| # | Теза |
|---|------|
| L1 | Панель дає лише **lastEvents (12)**. Повний розріз по парах/годинах/`exitReason` треба з `testbot-trades.jsonl`. |
| L2 | Confound: можливо, `stop_usd` спрацьовує швидше за pip-SL через розмір лота + спред → «миттєвий» мінус. Invert це не лікує сам по собі, якщо **виходи асиметричні**. |
| L3 | Два wins можуть бути шумом або інший `exitReason` (`target_usd` / `partial`). Перед prod-invert — порахувати **частку stop_usd серед exits**. Якщо ≈100% — теза анти-сигналу сильна. |
| L4 | Equity $1000 на панелі — **конфіг**, не live equity після −$739. Для рішень використовувати journal PnL, не UI Equity. |

### 3. SAGE (Risk)
> *«Динаміка ставок = скільки ризикуємо на угоду і скільки слотів. Не чіпаємо. Чіпаємо — чи взагалі входимо і куди дивиться ордер.»*

| Дозволено | Заборонено (під «динаміка ставок») |
|-----------|-----------------------------------|
| `FX_TESTBOT_INVERT_DIRECTION=1` | Зміна `riskPerTradePct` / units |
| Фільтр skip (не брати сигнал) | Martingale / double-after-loss |
| Симетризація TP/SL у **$** при тому ж risk% | Підняття `maxOpen` «щоб відігратись» |
| Session / streak pause | Зміна cooldown щоб «частіше стріляти» |

| # | Теза |
|---|------|
| S1 | Навіть після invert: **денний kill-switch** по sim P/L (напр. −$30/день) — це ризик-стоп, не сайзинг. |
| S2 | Max SL на UI зараз **$1+comm**, фактичні лоси **~$4** — розсинхрон конфігу/відображення. Вирівняти, інакше рішення по $ — сліпі. |

### 4. FLUX (Signal)
> *«Conviction 69–78 програє так само, як 60. Поріг score не врятує.»*

| # | Теза |
|---|------|
| F1 | Піднімати `minScore` **без** інших змін — майже марно на цьому журналі. |
| F2 | Два шляхи з тим самим сайзом: **(A) invert усіх qualifying**, **(B) брати лише subset** (пара/година/режими), де historical WR після invert або без нього > BE. |
| F3 | Charlie `action=SKIP` + `setupDraft` — testbot і так їсть draft. Invert draft = інверсія вже «заблокованого» math-gate сигналу; окремо заміряти draft vs live action. |

### 5. FORGE (Exits)
> *«Головний вбивця expectancy — не лише напрямок, а +$1 vs −$4.»*

| # | Теза |
|---|------|
| X1 | При фіксованому risk% симетризація виходів **не змінює stake**, змінює payoff: наприклад target **$3** / maxStop **$3**, або target **$2** / stop **$2**. |
| X2 | Комбо: **invert + симетричний $R:R (1:1 або 1.5:1)**. Invert дає WR; симетрія дає виживання при регресії WR з 99% до 55%. |
| X3 | Partial +$0.5 / 10хв майже не світиться в lastEvents → або не досягається, або серія занадто коротка до `stop_usd`. Після invert partial може почати спрацьовувати — логувати окремо. |

### 6. PIXEL (Meta-label)
> *«Не міняй розмір ставки — навчись казати NІ.»*

| # | Теза |
|---|------|
| P1 | Бінарна модель take/skip на фічах entry (pair, hour UTC, conv, side, spread, ATR) по історичному journal. Сайз завжди той самий. |
| P2 | Мінімум для v1 без ML: **blacklist пар** з найгіршим net після 20+ угод; **whitelist** топ-3. |
| P3 | Shadow: логувати «взяли б / не взяли б» при новому правилі, не лише executed — інакше знову сліпий тюнінг. |

### 7. PRISM (Timing)
> *«Останні угоди — 23:45–23:55. Це не kill-zone, це нічний шум.»*

| # | Теза |
|---|------|
| T1 | Без зміни сайзу: **вікно годин** (напр. лише 07:00–16:00 UTC). Менше угод, той самий $ на угоду. |
| T2 | Якщо більшість −$738 з off-hours — один фільтр годин може дати більший lift, ніж тюнінг score. Перевірити histogram по UTC hour на повному jsonl. |

### 8. ECHO (Regime)
> *«Коли всі пари conv=0 — ринок не дає setup. Не вигадуй входи.»*

| # | Теза |
|---|------|
| E1 | Нульова conv сьогодні — не баг панелі, а «немає кваліфікації». Після flip-стратегії все одно **не форсувати** входи. |
| E2 | News/blackout лишити; додати «streak pause»: 5 `stop_usd` поспіль → пауза 60хв (сайз не росте). |

## Раунд 2 — Суперечка

**NOVA:** Invert зараз — найдешевший flip expectancy при забороні міняти stakes.

**FORGE:** Invert при R:R 1:4 — це «вигравати часто по $1, інколи віддавати $4». Один поганий день після зміни режиму ринку з’їсть тиждень. Спочатку **вирівняти $TP/$SL**, потім invert.

**FLUX:** Якщо вирівняти виходи без invert при WR 0.8% — все одно смерть. Спочатку polarity, потім R:R.

**LENS:** Обидва праві в послідовності **вимірювання**:  
1) повний journal → % `stop_usd`, PnL by pair/hour;  
2) offline invert replay;  
3) offline invert + symmetric exits;  
4) лише потім `FX_TESTBOT_INVERT_DIRECTION=1` на sim.

**PIXEL:** Invert усіх = грубий баг-фікс сигналу. Додай meta-skip, інакше знову перекотимось у мінус, коли CHARLIE «виправиться» і антикореляція зникне.

**SAGE:** Якщо CHARLIE стане правим, invert стане отрутою. Тому invert лише з **авто-виміром rolling WR 50 угод**: якщо WR після invert < 45% — auto-disable invert.

**ORACLE:** Консенсус нижче.

## Раунд 3 — Як використати статистику (практика)

### Крок A — Розкласти існуючий журнал (сьогодні)

```
testbot-trades.jsonl
  → exitReason counts
  → pnl distribution
  → by pair: n, wr, net
  → by hour UTC: n, net
  → hold time (openedAt→closedAt): median до stop_usd
  → score buckets: 60–69 / 70–79 / 80+
```

Це і є «використати статистику»: не інтуїція, а таблиця, де **залишити / інвертувати / заборонити**.

### Крок B — Offline replay (без зміни stakes)

| Сценарій | Що міняємо | Що не чіпаємо |
|----------|------------|---------------|
| B0 Baseline | — | units, risk% |
| B1 Invert only | side flip | units, risk% |
| B2 Symmetric exits | targetUsd = maxStopUsd (напр. 3/3) | units, risk% |
| B3 Invert + symmetric | flip + 3/3 | units, risk% |
| B4 B3 + hour filter | + UTC window | units, risk% |
| B5 B3 + pair whitelist | + top pairs | units, risk% |

Метрики: net PnL, WR, PF, max DD, #trades (може впасти — це ок).

### Крок C — Live sim A/B (той самий сайзинг)

1. Увімкнути переможний сценарій з B* через env (invert / TP / SL $ / hour).  
2. **Не** чіпати `FX_RISK_PER_TRADE_PCT` / equity sizing.  
3. Rolling 50 trades: якщо PF < 1.0 — rollback.

## Фінальний консенсус ORACLE

### Що статистика вже доводить
1. При поточному payoff **+$1 / −~$4** система **не може** бути плюсовою без WR ≫ 75%.  
2. Факт WR **≪ 1%** ⇒ напрямок (або миттєвий вихід) **системно проти ринку**.  
3. Conviction **не predictive** на цьому режимі testbot.  
4. Найдешевший важіль **без зміни динаміки ставок** — **інверсія полярності** + **симетризація $TP/$SL** + **селекція (години/пари)**, не «ставити більше/менше».

### План «перевернути картинку» (пріоритет)

| P | Дія | Env / механізм | Зміна ставок? |
|---|-----|----------------|---------------|
| **P0** | Повний розбір jsonl (pair/hour/reason) | скрипт по `data/testbot-trades.jsonl` | ні |
| **P1** | Offline invert replay → очікуваний знак PnL | ноутбук/скрипт | ні |
| **P2** | `FX_TESTBOT_INVERT_DIRECTION=1` на sim | вже є в коді | ні |
| **P3** | Симетрія виходів: `TARGET_USD` ≈ `MAX_STOP_USD` (напр. 3 і 3) | env | ні (лише payoff) |
| **P4** | Hour filter + pair blacklist з журналу | фільтр перед entry | ні (лише частота) |
| **P5** | Auto-guard: rolling WR/PF вимикає invert | supervisor | ні |

### Чого НЕ робити
- Не піднімати сайз / maxOpen, щоб «відіграти −$738».  
- Не вірити minScore↑ як єдиному фіксу.  
- Не переносити invert на CHARLIE live, поки sim P1–P3 не дасть PF > 1.2 на 100+ нових угод.

### Одностроковий вердикт
> **Статистика каже: торгуєте дзеркало ринку з поганим R:R. Не чіпайте розмір ставки — розверніть дзеркало (`invert`), вирівняйте $TP/$SL, відсічіть години/пари де журнал токсичний, і закріпіть rollback по rolling PF.**

## Додаток — формула для панелі / Telegram

```
BE_WR = avgLoss / (avgWin + avgLoss)
EDGE  = WR * avgWin - (1-WR) * avgLoss

Після invert (наївний replay):
WR' ≈ 1 - WR
avgWin' ≈ min(targetUsd, |old avgLoss|)   # часто ≈ targetUsd
avgLoss' ≈ maxStopUsd                      # колишні рідкісні wins
```

Якщо після P2+P3 на нових 100 sim-угах `EDGE ≤ 0` — повертатись до діагнозу (виконання/спред), не крутити risk%.
