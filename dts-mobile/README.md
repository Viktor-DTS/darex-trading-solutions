# DTS Mobile (Flutter)

Мобільний клієнт DTS Service для Android та iOS.

## Що реалізовано (MVP)

- **Авторизація**: логін, збереження токена та користувача (Secure Storage), AuthGate.
- **Головний екран**: модулі за роллю (Сервіс, Оператор, Склад, Тестування, Менеджери).
- **Сервіс**: список заявок, фільтр по статусу/пошук, деталі заявки, зміна статусу, звіт (роботи, матеріали, транспорт), завантаження фото (камера/галерея).
- **Оператор**: майстер створення заявки (клієнт, ЄДРПОУ, адреса, регіон, планова дата, контакт), чернетки, підтягування даних по ЄДРПОУ.
- **Склад**: обладнання, «в дорозі», підтвердження отримання, переміщення/відвантаження, документи переміщення та відвантаження, додавання обладнання, batch move/ship.
- **Тестування**: заявки на тестування, деталі, завершення тесту, фото.
- **Менеджери**: огляд заявок, статистика, деталі, таймлайн.

## Запуск

1. **Платформи** (якщо ще немає папок `android/` та `ios/`):
   ```bash
   cd dts-mobile
   flutter create . --platforms=android,ios
   ```

2. **API**: у `lib/core/config.dart` вказано базовий URL бекенду (без `/api` у кінці). За потреби змініть `AppConfig.apiBaseUrl` на свій сервер.

3. **Залежності та запуск**:
   ```bash
   flutter pub get
   flutter run
   ```

## Дозволи для фото (камера / галерея)

Після `flutter create .` додайте дозволи:

### Android

У файл `android/app/src/main/AndroidManifest.xml` всередині тега `<manifest>` (наприклад, перед `<application>`):

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" android:maxSdkVersion="32" />
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" />
```

### iOS

У `ios/Runner/Info.plist` додайте:

```xml
<key>NSCameraUsageDescription</key>
<string>ДТС потребує доступу до камери для фото звітів</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>ДТС потребує доступу до галереї для додавання фото</string>
```

## Оновлення додатку (Google Drive)

Додаток при вході перевіряє, чи є нова версія (бекенд `GET /api/app-version`). Якщо є — показується діалог «Є оновлення» з кнопкою **Оновити**, яка відкриває посилання на завантаження APK (наприклад, з Google Drive).

### Що зробити один раз (налаштування)

1. **Посилання на завантаження APK**  
   - Завантажте `app-release.apk` у папку на Google Drive (наприклад [APP Mobile](https://drive.google.com/drive/folders/1zCudgTg2zp7WlMBJHUC-PRsMrccs2dZZ?usp=drive_link)).  
   - Відкрийте файл **app-release.apk** → «Поділитися» → «Доступ для всіх, хто має посилання» → скопіюйте посилання.  
   - Для прямого завантаження змініть посилання так: якщо посилання має вигляд  
     `https://drive.google.com/file/d/XXXXX/view?usp=sharing`,  
     то посилання для завантаження:  
     `https://drive.google.com/uc?export=download&id=XXXXX`  
     (підставте замість `XXXXX` ID файлу з вашого посилання).

2. **Версія та посилання на APK** задаються у файлі бекенду (без змін на Render):  
   У репозиторії **darex-trading-solutions/backend** відкрийте `app-version.json` і при потребі вкажіть:
   - **latest_version** — та сама версія, що в `pubspec.yaml` (наприклад `0.1.0`).
   - **android_store_url** — посилання для завантаження APK у форматі `https://drive.google.com/uc?export=download&id=...`.
   За потреби: **min_version**, **force_update**, **ios_store_url**.  
   Після змін зробіть commit і push — після деплою на Render додаток буде брати ці значення.  
   (Змінні оточення на Render можна використовувати як запасний варіант, якщо файлу немає.)

Після цього при відкритті додатку користувачі зі старою версією побачать діалог оновлення і зможуть перейти по посиланню та завантажити новий APK.

### Що робити при кожному випуску нової версії

1. **Підняти версію в проєкті**  
   У `pubspec.yaml` змініть, наприклад:  
   `version: 0.1.0+1` → `version: 0.2.0+2` (перше число — версія для користувача, друге — build number).

2. **Зібрати APK**  
   ```bash
   cd dts-mobile
   flutter build apk
   ```  
   Файл: `build/app/outputs/flutter-apk/app-release.apk`.

3. **Завантажити в Google Drive**  
   Замініть старий **app-release.apk** у вашій папці новим файлом (або завантажте новий файл і оновіть посилання, якщо використовуєте окремий файл на кожну версію).

4. **Оновити версію на бекенді (тільки в коді)**  
   У репозиторії відкрийте **darex-trading-solutions/backend/app-version.json** і змініть **latest_version** на нове значення (наприклад `0.2.0`). Якщо змінили посилання на APK — оновіть **android_store_url**.  
   Зробіть commit і push бекенду — Render після деплою підхопить новий файл. **Заходити в Render і змінювати Environment не потрібно.**

Після цього при наступному відкритті додатку користувачі з версією нижчою за вказану побачать діалог оновлення і зможуть завантажити новий APK з Google Drive.
